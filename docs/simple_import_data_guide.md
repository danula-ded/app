# Простой import_data на 5 баллов

Файл:

```text
core/management/commands/import_data.py
```

Запуск:

```powershell
uv run python manage.py import_data
```

## Что важно для баллов

Покажи эксперту 4 вещи:

1. Это Django Management Command.
2. Она читает Excel через `openpyxl`.
3. Она использует `get_or_create()` и `update_or_create()`.
4. Повторный запуск не создает дубли.

## Как переписать под новое задание

Открываешь файл и меняешь только верхний блок:

```python
FILES = {...}
PRODUCT = {...}
USER = {...}
ORDER = {...}
ROLE_MAP = {...}
```

Если дали не товары, а услуги, внутреннее имя `PRODUCT` можно оставить. Это просто техническое имя главной сущности.

## Если Excel-файлов не 4, а 3

Ничего страшного. Команда проверяет наличие файла:

```python
self.run_if_exists(...)
```

Если файла нет, она напишет:

```text
Файл ... не найден, импорт пропущен
```

И продолжит работу.

## Что обычно менять

### Названия файлов

```python
FILES = {
    "points": "Пункты выдачи_import.xlsx",
    "products": "Tovar.xlsx",
    "users": "user_import.xlsx",
    "orders": "Заказ_import.xlsx",
}
```

### Номера колонок

Номера считаются с нуля:

```text
первая колонка  -> 0
вторая колонка  -> 1
третья колонка  -> 2
```

Пример:

```python
PRODUCT = {
    "article": 0,
    "name": 1,
    "price": 3,
}
```

### Роли

```python
ROLE_MAP = {
    "Администратор": "admin",
    "Менеджер": "manager",
    "Клиент": "client",
}
```

## Главное правило

Не пытайся сделать импорт универсальным для всех заданий.

На экзамене нужно быстро написать импорт под конкретные Excel-файлы. Этот шаблон помогает сделать это спокойно и одинаково каждый раз.
