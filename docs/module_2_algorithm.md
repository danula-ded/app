# Алгоритм разработки приложения. Модуль 2

Документ нужен для требования модуля 2: оформить алгоритм разработки приложения в виде блок-схемы по ГОСТ 19.701-90 и представить в PDF.

## Назначение алгоритма

Алгоритм описывает порядок создания веб-приложения ООО «Обувь» на Django: от подготовки проекта до проверки входа, списка товаров и ролей пользователей.

## Входные данные

- файлы Excel с товарами, пользователями, заказами и пунктами выдачи;
- ресурсы приложения: логотип, иконка, заглушка изображения;
- учетные данные PostgreSQL;
- техническое задание демоэкзамена.

## Выходные данные

- работающее Django-приложение;
- PostgreSQL-база с импортированными данными;
- страница входа;
- список товаров;
- интерфейсы гостя, клиента, менеджера и администратора;
- блок-схема в PDF.

## Основные шаги

1. Подготовить проект Django и зависимости в `pyproject.toml`/`uv.lock`.
2. Настроить подключение к PostgreSQL в `config/settings.py`.
3. Создать приложение `core`.
4. Описать модели предметной области в `core/models.py`.
5. Создать и применить миграции.
6. Написать management command `import_data`.
7. Импортировать Excel-данные в БД.
8. Настроить авторизацию.
9. Создать страницу списка товаров.
10. Добавить проверку ролей.
11. Подключить статические файлы и медиа.
12. Проверить проект командой `python manage.py check`.
13. Запустить сервер и проверить страницы в браузере.

## Блок-схема

PDF-файл лежит рядом:

```text
docs/module_2_algorithm_gost.pdf
```

В блок-схеме используются стандартные обозначения:

- начало/конец;
- процесс;
- ввод/вывод данных;
- условие;
- переход по стрелке.
