# Универсальный алгоритм разработки

Этот алгоритм подходит для типового задания демоэкзамена, даже если меняется тема: магазин, шиномонтаж, услуги, склад, заявки.

Документ с блок-схемой:

```text
docs/universal_development_algorithm_gost.pdf
```

## Назначение

Описать общий порядок разработки программного модуля по техническому заданию.

## Входные данные

- текст задания;
- Excel-файлы для импорта;
- ресурсы интерфейса;
- требования к ролям;
- требования к базе данных;
- критерии оценивания.

## Выходные данные

- база данных;
- приложение;
- импорт данных;
- пользовательские интерфейсы;
- проверенные роли;
- документация;
- результат в репозитории.

## Шаги алгоритма

1. Начало.
2. Получить и прочитать задание.
3. Выделить сущности, поля, связи и роли.
4. Спроектировать базу данных в 3НФ.
5. Создать проект и установить зависимости.
6. Реализовать модели и миграции.
7. Настроить PostgreSQL.
8. Реализовать импорт данных из Excel.
9. Реализовать авторизацию и разграничение ролей.
10. Реализовать список главной сущности.
11. Реализовать поиск, фильтр и сортировку.
12. Реализовать добавление, редактирование и удаление.
13. Реализовать работу с заказами/заявками.
14. Настроить CSRF, медиа и проверку изображений.
15. Проверить приложение.
16. Подготовить скриншоты и репозиторий.
17. Сформировать SQL-скрипт и ER-диаграмму средствами DBeaver.
18. Конец.

## Фраза для защиты

Алгоритм начинается с анализа задания и проектирования базы данных. Затем создается Django-проект, описываются модели, выполняются миграции и импорт данных. После этого реализуются пользовательские интерфейсы, роли, CRUD-операции и проверка работы приложения. В конце формируются документы и результат загружается в репозиторий.
