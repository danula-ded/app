# Демоэкзамен 2026. ООО Обувь

Простой Django-проект под требования демоэкзамена: PostgreSQL, импорт Excel, роли, товары, заказы, CRUD, CSRF, изображения и очистка старых файлов.

## Быстрый запуск

```powershell
uv sync
uv run python manage.py migrate
uv run python manage.py import_data
uv run python manage.py runserver
```

Открыть:

```text
http://127.0.0.1:8000/
```

## База данных

По умолчанию используется PostgreSQL:

```text
database: demoexam_26
user: postgres
host: localhost
port: 5432
```

Пароль лежит в `config/settings.py`. На экзамене его нужно заменить на свой пароль PostgreSQL.

## Пользователи

Пользователи импортируются из `core/import/user_import.xlsx`.

Роли:

```text
admin   - товары и заказы: просмотр, добавление, редактирование, удаление
manager - товары: просмотр с поиском/фильтрацией/сортировкой; заказы: просмотр
client  - товары: только просмотр
guest   - товары: только просмотр без входа
```

## Где менять проект, если на экзамене другие данные

Сначала открывай:

```text
core/exam_config.py
```

Там лежат:

- названия ролей;
- поля поиска;
- варианты сортировки;
- размер фото товара.

Если поменялись Excel-файлы или номера колонок, открывай:

```text
core/management/commands/import_data.py
```

В самом верху там есть блок:

```text
МЕНЯТЬ ТОЛЬКО ЭТОТ БЛОК
```

Если поменялась тема, например вместо магазина обуви будет шиномонтажка, смотри:

```text
docs/adapt_to_new_theme.md
```

Если появились новые таблицы или полностью другая предметная область, тогда нужно менять уже не только конфиг, а еще:

```text
core/models.py
core/forms.py
core/views.py
core/templates/core/
```

Это нормально: в Django структура базы данных всегда начинается с моделей.

Модуль 1:

- модели в 3НФ;
- внешние ключи;
- PostgreSQL;
- импорт данных из Excel;
- idempotent-импорт через `update_or_create`.

Модуль 2:

- окно входа;
- вход гостем;
- список товаров;
- логотип, иконка, единый стиль;
- фото товара или заглушка;
- подсветка скидки и отсутствия на складе.

Модуль 3:

- поиск по текстовым полям;
- фильтр по поставщику;
- сортировка по количеству;
- добавление/редактирование/удаление товаров;
- запрет удаления товара, если он есть в заказе;
- CSRF во всех формах;
- кнопки CRUD видны только администратору.

Модуль 4:

- список заказов для менеджера и администратора;
- добавление/редактирование/удаление заказов для администратора;
- товары заказа вводятся строкой: `А112Т4, 2, F635R4, 2`.

Вариативная часть 25 баллов:

- `uv.lock` фиксирует библиотеки;
- импорт Excel добавляет и обновляет данные без дублей;
- связи в PostgreSQL сохраняются;
- роли ограничивают доступ к интерфейсу;
- `Pillow` проверяет и сжимает изображения до 300x200;
- старые изображения удаляются при замене или удалении товара.
